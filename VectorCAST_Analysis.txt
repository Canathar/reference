   Determining VectorCAST Unit Test Environment Properties
   ═══════════════════════════════════════════════════════
      1. Open a terminal window as a non-root user
      2. Change to your local source code repository VectorCAST directory by typing the following command(s) into the terminal window:

            cd <local-src-directory>/ccu/base/tools/vcast/VCS21/environment/<unit-test-environment>
            (ex: /data/dataworkspace/trunk/ccu/base/tools/vcast/VCS21/environment/APPI_AG_COMMONGRSENDPOINT)

      3. Search for the total number of functions with unit tests by typing the following command(s) into the terminal window:

            grep -rnw . -e '-- Subprogram' | wc -l

      4. Search for the total number of test cases written by typing the following command(s) into the terminal window:

            grep -rnw . -e '-- Test Case:' | wc -l

      5. Search for the total number of auto-generated test cases written by typing the following command(s) into the terminal window:

            grep -rnw . -e '-- Test Case: BASIS' | wc -l

      6. Search for the total number of manual test cases written by typing the following command(s) into the terminal window:

            grep -rnw . -e '-- Test Case:' | grep -v 'BASIS' | wc -l


   Analyze The Build Output
   ════════════════════════
      1. Open a terminal window as a non-root user
      2. Run the script to generate the unit test environment analysis by typing the following command(s) into the terminal window:

            ./analyze-unit-test-environments.sh > <unit-test-environment-results-file>
            (ex: ./analyze-unit-test-environments.sh > target_results.txt)

      3. Search for the total number of failed environments by typing the following command(s) into the terminal window:

            grep <unit-test-environment-results-file> -e 'Error' -e 'Failed' | sed '/^\s*$/d' | wc -l
            (ex: grep native_results.txt -e 'Error' -e 'Failed' | sed '/^\s*$/d' | wc -l)
            (ex: grep target_results.txt -e 'Error' -e 'Failed' | sed '/^\s*$/d' | wc -l)

      4. Display the details for the failed environments by typing the following command(s) into the terminal window:

            grep -rw -A 8 -B 4 <unit-test-environment-results-file> -e 'Error' -e 'Failed'
            (ex: grep -rw -A 8 -B 4 native_results.txt -e 'Error' -e 'Failed')
            (ex: grep -rw -A 8 -B 4 target_results.txt -e 'Error' -e 'Failed')
